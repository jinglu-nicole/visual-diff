# 🎮 Visual Diff · 游戏美术还原度检查工具

上传 UI 设计效果图与游戏实机截图，AI 自动逐组件对比分析，输出结构化还原度评估报告。

![纯前端架构](https://img.shields.io/badge/架构-纯前端_无后端-blue) ![Claude Vision](https://img.shields.io/badge/AI-Claude_Vision-orange) ![部署](https://img.shields.io/badge/部署-Netlify_/_静态托管-green)

---

## ✨ 功能亮点

- **8 维度评分** — 位置、大小、间距、字体、色彩、图标、背景、视觉层级全面覆盖
- **递归组件树** — 至少 3 层拆解 UI 结构，逐组件逐状态对比
- **严重度分级** — 🔴 高 / 🟡 中 / 🟢 低 三级问题标注，支持筛选
- **纯前端架构** — 浏览器直接调用 Claude API，无需搭建后端
- **历史记录** — 分析结果自动保存到浏览器本地，随时回看
- **链接分享** — 每次分析生成唯一 URL（`#/task/xxx`），刷新不丢失

## 📖 使用说明

### 第一步：配置 API

打开工具后，在顶部配置栏填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| **API Key** | Claude API 密钥 | `sk-ant-...` |
| **服务商 URL** | API 服务地址 | `https://api.anthropic.com` 或内部代理地址 |

> 💡 API Key 和服务商 URL 会自动保存到浏览器本地，下次打开无需重新输入。

点击右侧 ⌄ 展开更多设置（通常不需要修改）：

| 高级设置 | 默认值 | 说明 |
|----------|--------|------|
| Thinking 预算 | $0.18 | AI 深度思考的 token 预算，越高分析越细致 |
| 画布宽度 | 2100 | 设计稿的基准分辨率宽度（px） |
| 画布高度 | 1080 | 设计稿的基准分辨率高度（px） |

### 第二步：上传图片

左右两个上传区域：

| 区域 | 用途 | 说明 |
|------|------|------|
| **左侧 · 设计稿** | 目标效果 | UI 设计效果图（来自 Figma/PS 等） |
| **右侧 · 实机截图** | 实际还原 | 游戏引擎中的实机截图 |

三种上传方式：
1. **拖放** — 直接拖图片到对应区域
2. **粘贴** — 点击区域后 `Ctrl+V` 粘贴截图
3. **点击** — 点击区域弹出文件选择器

> ⚠️ 两张图片应是**同一个界面**的设计稿和实现截图。文本内容可以不同（如玩家名字、数值），工具只对比视觉样式。

### 第三步：开始分析

点击「开始分析」按钮，等待 AI 完成分析（通常 30-120 秒）。

分析过程中会显示 4 个阶段的进度：
1. 🖼️ **图片预处理** — 编码图片、提取主色调
2. 📡 **发送请求** — 将图片发送到 Claude API
3. ⏳ **等待响应** — AI 深度分析中（最耗时的阶段）
4. 📋 **解析结果** — 整理输出报告

### 第四步：查看报告

分析完成后，报告分为左右两栏：

#### 左栏 · 组件树

按层级展示识别到的所有 UI 组件：
```
▌右下 · 奖励区域
  └ 奖励标题
  └ 奖励列表
    ◦ 奖励项（默认态）
      · 奖励图标
      · 奖励名称
      · 奖励数量
```

#### 右栏 · 问题清单 + 评分

**问题清单**按组件归组，每条问题标注：
- **严重度**：🔴 高 / 🟡 中 / 🟢 低
- **维度**：P1 位置 / P2 大小 / P3 间距 / P4 字体 / P5 色彩 / P6 图标 / P7 背景 / P8 层级
- **描述**：具体差异说明

**还原度评分**以表格形式展示 8 个维度的打分（每项 /10）和综合得分（/100）。

#### 严重度筛选

报告标题右侧有 🔴🟡🟢 三个筛选按钮，点击可按严重程度过滤问题清单。

## 🕐 历史记录

- 每次分析完成后，结果会**自动保存**到浏览器 localStorage
- 在首页下方可以看到**历史记录列表**，显示缩略图、时间和问题统计
- 点击任意历史记录即可回看完整报告（含设计稿/截图预览）
- URL 会自动变为 `#/task/xxx`，可以收藏或刷新页面恢复查看
- 鼠标悬停在历史记录上，右侧会出现删除按钮 🗑️
- 最多保留 50 条历史记录

## 🚀 部署

本工具是**纯前端应用**，构建后只有静态文件，可部署到任何静态托管服务。

### Netlify / Vercel / 静态服务器

```bash
# 安装依赖
cd frontend && npm install

# 构建
npm run build

# 产物在 frontend/dist/ 目录
```

将 `frontend/dist/` 目录部署到任意静态托管即可。

### 本地开发

```bash
cd frontend
npm install
npm run dev
# 访问 http://localhost:5173
```

## ❓ 常见问题

### Q: 分析时报"跨域访问被拦截"？
A: 浏览器直连 Claude API 需要服务端支持 CORS。如果使用官方 `api.anthropic.com`，需要在请求头中包含 `anthropic-dangerous-direct-browser-access: true`（工具已自动添加）。如果使用内部代理，请确认代理支持 CORS。

### Q: 分析很慢，需要多久？
A: 通常 30-120 秒。AI 需要仔细分析两张图片的每个组件，Thinking 预算越高，分析越细致但耗时更长。

### Q: 可以对比非游戏 UI 吗？
A: 可以，但 Prompt 针对游戏 UI 的锚点适配、九宫格等特性做了优化，对游戏 UI 效果最好。

### Q: 历史记录存在哪里？
A: 浏览器 localStorage，清除浏览器数据会丢失。不同浏览器/设备的历史不共享。

### Q: API Key 安全吗？
A: API Key 仅保存在你的浏览器 localStorage 中，所有请求从你的浏览器直接发送到 API 服务商，不经过任何第三方服务器。
